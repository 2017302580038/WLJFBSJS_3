##### Enable Telnet on Windows

1. Open the Control Panel

2. Go to Programs & Features

3. In left bar select "Turn Windows features on or off"

4. Find "Telnet Client" and tick it

5. Click "OK"

```py
import base64

message = input()

print(base64_message)
```

### Question 1

> **P23.** Consider distributing a file of F bits to N peers using a **client-server** architecture. Assume a fluid model where the server can simultaneously transmit to multiple peers, transmitting to each peer at different rates, as long as the combined rate does not exceed u<sub>s</sub>.

> **a.** Suppose that u<sub>s</sub>/N <= d<sub>min</sub>. Specify a distribution scheme that has a distribution time of NF/u<sub>s</sub>.

>> **My Answer:**
>>
>> If the server transmit the file to each peer at the same rate, which is u<sub>s</sub> / N, since u<sub>s</sub> / N <= d<sub>min</sub> which is the minimum downloading rate of each peer, the peers are all download the file at rate u<sub>s</sub> / N. So, using this distribution scheme, we get a distribution time of F / (u<sub>s</sub> / N) = NF/u<sub>s</sub>.

> **b.** Suppose that u<sub>s</sub>/N >= d<sub>min</sub>. Specify a distribution scheme that has a distribution time of F/d<sub>min</sub>.

>> **My Answer:**
>>
>> If the server transmit the file to each peer at the same rate d<sub>min</sub>, as u<sub>s</sub>/N >= d<sub>min</sub>, we could see that the combined rate is d<sub>min</sub> x N <= u<sub>s</sub>/N x N = u<sub>s</sub>, which does not exceed u<sub>s</sub>.
>>
>> So, in this circumstance, we get a distribution time of F / d<sub>min</sub>.

> **c.** Conclude that the minimum distribution time is in general given by max{NF/u<sub>s</sub>, F/d<sub>min</sub>}.

>> **My Answer:**
>>
>> When u<sub>s</sub>/N <= d<sub>min</sub>, the combined rate is u<sub>s</sub>, as it cannot exceed u<sub>s</sub>, if the server does not transmit the file to each peer at the same rate u<sub>s</sub> / N, there must be a peer which downloads the file at rate < u<sub>s</sub> / N, which makes the distribution time > NF/u<sub>s</sub>.
>>
>> When u<sub>s</sub>/N > d<sub>min</sub>, there must be a peer which downloads the file at rate d<sub>min</sub>, so the distribution time must >= F / d<sub>min</sub>.
>>
>> So, the minimum distribution time is in general given by max{NF/u<sub>s</sub>, F/d<sub>min</sub>}.

### Question 2

> **P24.** Consider distributing a file of F bits to N peers using a **P2P** architecture. Assume a fluid model. For simplicity assume that d<sub>min</sub> is very large, so that peer download bandwidth is never a bottleneck.

> **a.** Suppose that u<sub>s</sub> <= (u<sub>s</sub> + u<sub>1</sub>+...+u<sub>N</sub>) / N. Specify a distribution scheme that has a distribution time of F/u<sub>s</sub>.

>> **My Answer:**
>>
>> Let u = u<sub>1</sub> + ... + u<sub>N</sub>. If the server transmit F * (u<sub>i</sub> / u) bits of the file content to peer i at rate u<sub>s</sub> * (u<sub>i</sub> / u), and let each peer forward the file content they received from server to other peers at rate u<sub>s</sub> * (u<sub>i</sub> / u), we could see in this circumstance that for peer i, the combined rate is u<sub>s</sub> * (u<sub>i</sub> / u) * (N - 1) <= (u<sub>s</sub> + u) / N * u<sub>i</sub> / u * (N - 1) = u<sub>i</sub> * (u<sub>s</sub> + u) / u * (N - 1) / N <= u<sub>i</sub>.
>>
>> In this circumstance, each peer's downloading rate is u<sub>s</sub>. So, we get a distribution time of F/u<sub>s</sub>.

> **b.** Suppose that u<sub>s</sub> >= (u<sub>s</sub> + u<sub>1</sub>+...+u<sub>N</sub>) / N. Specify a distribution scheme that has a distribution time of NF/(u<sub>s</sub>+u<sub>1</sub>+…+u<sub>N</sub>).

>> **My Answer:**
>>
>> Let u = u<sub>1</sub> + ... + u<sub>N</sub>. First of all, we break file into N+1 parts, for peer i, we transmit part i to it at rate r<sub>i</sub> = u<sub>i</sub> / (N-1). Meanwhile, peer i forwards the bits it receives to other N-1 peers. For part N+1, it is transmitted by the server directly to each peer at rate r<sub>N+1</sub> = (u<sub>s</sub>-u/(N-1))/N, so that the server's combined rate reaches u<sub>s</sub>. (Since u<sub>s</sub> >= (u<sub>s</sub> + u<sub>1</sub>+...+u<sub>N</sub>) / N, r<sub>N+1</sub> must >= 0)
>>
>> In this circumstance, each peer's downloading rate is u / (N-1) + (u<sub>s</sub>-u/(N-1))/N = (u<sub>s</sub> + u) / N.
>>
>> So, we get a distribution time of F / ((u<sub>s</sub> + u) / N) = NF/(u<sub>s</sub>+u<sub>1</sub>+…+u<sub>N</sub>).

> **c.** Conclude that the minimum distribution time is in general given by max{F/u<sub>s</sub>,NF/(u<sub>s</sub>+u<sub>1</sub>+…+u<sub>N</sub>)}.

>> **My Answer:**
>>
>> When u<sub>s</sub> <= (u<sub>s</sub> + u<sub>1</sub>+...+u<sub>N</sub>) / N, each peer cannot download file at rate > u<sub>s</sub>, since the server cannot transmit at rate > u<sub>s</sub>, so the minimum distribution time is F/u<sub>s</sub>.
>>
>> For each peer i, if any r<sub>j(in)</sub> (1 <= j <= N+1) increases, then for peer j, there must be some r<sub>k(out)</sub> decreases since the combined rate of peer j should not > u<sub>j</sub>. So, for peer k, it will get part j of the file slower than before. Thus increase the distribution time.
>>
>> So, the minimum distribution time is in general given by max{F/u<sub>s</sub>,NF/(u<sub>s</sub>+u<sub>1</sub>+…+u<sub>N</sub>)}.
